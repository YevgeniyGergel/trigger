## ADDED Requirements

### Requirement: Notification Channel Configuration
The system SHALL allow a psychologist to configure which notification channels (email, Telegram) are used to notify them and their clients about bookings, cancellations, reminders, and payments.

#### Scenario: Default channel on registration
- **WHEN** a psychologist registers
- **THEN** the system enables email notifications by default and offers Telegram as an optional additional channel

### Requirement: Telegram Account Linking
The system SHALL allow a psychologist or client to link their Telegram account to receive notifications, via a one-time deep-link generated by the system.

#### Scenario: Client links Telegram during booking
- **WHEN** a client opens the Telegram deep-link shown after completing a booking
- **THEN** the system associates that client's Telegram chat ID with their client record and confirms the link in the chat

#### Scenario: Unlinked client falls back to email
- **WHEN** a client has not linked Telegram and provided only an email address
- **THEN** the system sends all notifications for that client via email only

### Requirement: Booking Confirmation Notification
The system SHALL notify both the psychologist and the client through their configured channels when a session is booked.

#### Scenario: New booking notifies psychologist
- **WHEN** a client successfully books a session via the public booking page
- **THEN** the system sends a notification to the psychologist with the client's name and requested time

#### Scenario: New booking confirms to client
- **WHEN** a booking is created
- **THEN** the system sends a confirmation notification to the client with session date, time, and status

### Requirement: Session Reminder Notification
The system SHALL send an automated reminder to the client a configurable number of hours before a confirmed session.

#### Scenario: Reminder sent before session
- **WHEN** the configured reminder lead time before a confirmed session is reached
- **THEN** the system sends a reminder notification to the client through their configured channel(s)

#### Scenario: Cancelled session does not trigger reminder
- **WHEN** a session is cancelled before the reminder lead time is reached
- **THEN** the system does not send a reminder for that session

### Requirement: Payment Status Notification
The system SHALL notify the psychologist and the client when a session's payment status changes to paid or failed.

#### Scenario: Payment succeeds
- **WHEN** a session's payment status changes to "paid"
- **THEN** the system notifies both the client and the psychologist of the successful payment

#### Scenario: Payment fails
- **WHEN** a session's payment status changes to "failed"
- **THEN** the system notifies the client with a retry link and notifies the psychologist of the failed attempt

### Requirement: Email Fallback Guarantee
The system SHALL always deliver notifications via email even if Telegram is configured, unless the psychologist explicitly disables email for that notification type.

#### Scenario: Telegram delivery fails
- **WHEN** a Telegram notification fails to deliver (e.g., bot blocked by user)
- **THEN** the system still delivers the notification via email if an email address is on file
